## 2. Simulation Bypass Safety Audit
We conducted a complete safety audit of the simulated outbound behavior to ensure no real message failures are silently ignored in production. 
- **Previous Risk:** If Meta credentials were not set in the `.env` file, the send workers would automatically skip the API call and mark the message as `SENT`. This meant a simple misconfiguration in production could silently drop messages.
- **Fixed:** Workers no longer rely on missing environment variables to activate simulation mode.

## 3. Explicit Simulation Marker Requirement
- Outbound messages are now **only** bypassed if they carry an explicit simulation marker (`isSimulation: true`) within their BullMQ job payload.
- This marker is generated by `InteractiveCaptureService` **only** if the inbound webhook event was tagged with `is_simulation: true` by the manual QA scripts.

## 4. Confirmation: Missing Meta Secrets Do Not Silently Mark Real Messages SENT
- **Confirmed.** If a real outbound message is enqueued (i.e. no simulation marker) and the Meta credentials (`INSTAGRAM_APP_SECRET` or `META_APP_SECRET`) are missing, the send workers will now accurately log an error and mark the message status as `FAILED`.
- They will throw an explicit error indicating missing credentials, preventing silent drops and allowing the job to be retried safely once credentials are provided.
